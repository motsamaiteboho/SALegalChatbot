How RAG Works
RAG explained

Retrieval-Augmented Generation (RAG) combines local document retrieval with a language model answer.

Step 1: Embed the question

The question is converted into a numeric embedding that captures meaning.

Step 2: Search the vector store

The embedding is used to find similar judgment fragments in the FAISS vector store.

Step 3: Build the context

Retrieved case excerpts and metadata are assembled into a context prompt.

Step 4: Generate the answer

The model writes a South African legal answer grounded in the retrieved cases.

Mode comparison

Case-Law mode uses retrieved SAFLII context. General mode uses broader LLM reasoning without local retrieval.

Glossary
  • Embeddings: numeric vectors representing text meaning.
  • Vector store: a searchable database of embeddings.
  • Retriever: finds the most relevant documents by similarity.
  • Context: retrieved text used to ground the answer.
  • RAG: retrieval-augmented generation.
